Big Questions

ROEs
- Avoid Vendor lock
- Open-source AMAP
- Avoid package/library/api dependencies (but also dont recreate the wheel)
- Be excellent to each other


Architecture

	- Do we build this for primarily web-services? (REST/Grpc)
	- Or do we build this with IOT in mind? (GRPC, pub/sub, mqtt)
		- if yes then do we need to consider RUST
	- Who are the primary users?
	- Where does CTO want to move with regards to IAM, Authorization, Authentication (this is going to be the foundation on which we build this architecture)
		notes: I suggest that the default is the agentic pathways use the active users credentials, the agents themselves should have not creds but assume the identity of the user
	- What networks does this need to ride on (currently i dont think there is much azure on yellow so we would be dealing with azure on red and AWS on yellow)
